# Handoff: くるみ 学習ダッシュボード（中学受験・小学生向け）

## Overview
小学生（中学受験）の学習進捗を管理するダッシュボードです。本人・保護者が、教材（テキスト／テスト）の進捗、間違い直し、日々の学習予定と報告、科目別の学習バランスを一目で確認できます。

対象科目は **算数・国語・理科・社会の4科のみ**。データは **Obsidian の保管庫（Markdownノート）に入力 → パイプラインで吸い上げ → ダッシュボードに表示** という構成を想定しています（後述「データパイプライン」）。

---

## About the Design Files
このバンドル内の `README.md` は **HTMLで作られたデザイン参照（プロトタイプ）の仕様書** です。プロトタイプ本体は、このREADMEの**1つ上の階層（同梱キット `kurumi-dashboard/` 直下）**にある一連のファイル（`index.html`・`*.jsx`・`data.js`）です。見た目と挙動の意図を示すもので、そのまま本番投入するコードではありません。

タスクは、これらのHTMLデザインを **対象コードベースの環境（React/Next.js 等）で、その既存パターン・ライブラリを使って再現する**ことです。まだ環境がない場合は、プロジェクトに最適なフレームワークを選んで実装してください。プロトタイプは React + Babel(standalone) + 純CSSトークンで書かれていますが、本番ではビルド済みのReact/TypeScript構成への移植を推奨します。

ロジック（集計関数・カレンダー生成）は `../data.js`（同梱キット直下）にあり、ほぼそのまま移植できます。

> **動かし方**: 同梱キットの `index.html` をブラウザで開けばプロトタイプが動きます（CSS・トークン・コンポーネントは同じ `kurumi-dashboard/` 内および上位の `tokens/`・`styles.css` を参照）。

## Fidelity
**High-fidelity (hifi)。** 最終的な配色・タイポグラフィ・余白・インタラクションまで作り込んであります。UIはこの見た目を忠実に再現してください（後述「Design Tokens」に値を記載）。データはすべて仮（モック）です。

## データパイプライン（このプロジェクト特有・最重要）

```
┌─────────────────┐     ┌──────────────────────┐     ┌─────────────────┐
│ Obsidian 保管庫  │ →  │ パイプライン           │ →  │ ダッシュボード   │
│ (Markdownノート) │     │ frontmatter をパース   │     │ (このデザイン)   │
│  ・教材ノート     │     │ → 集計 → data.json     │     │ data.json を読む │
│  ・テスト結果     │     │ (build時 or watch)    │     │                 │
│  ・間違いノート   │     └──────────────────────┘     └─────────────────┘
│  ・日々の報告     │
└─────────────────┘
```

### 推奨実装
- **パーサ**: Node スクリプト。`gray-matter`（frontmatterパース）＋ `glob` で保管庫の `.md` を走査し、`data.json` を生成。
- **実行**: 開発時は `chokidar` でファイル監視してホットリロード。本番は Obsidian の Git 同期 → CI で `data.json` を生成 → 静的サイトを再デプロイ、あるいはサーバー側で都度読込。
- **集計ロジック**: `../data.js`（同梱キット直下の `data.js`）の `window.subjectStats(...)` と、IIFEで生成しているカレンダー `plan` / `reports` の処理をそのまま TypeScript 関数に移植する。生データ（教材・テーマの done/total、予定、報告）はObsidianから、派生値（%・残り問数・冊数）はパイプラインで計算。

### Obsidian ノート設計（frontmatter スキーマ案）
保管庫内をフォルダで分け、frontmatter（YAML）に構造データ、本文にメモ・写真を置く想定。

**教材ノート** `教材/プラスワン問題集.md`
```yaml
---
type: text            # text | test
subject: 算数          # 算数 | 国語 | 理科 | 社会
title: プラスワン問題集
publisher: 東京出版
material_type: book   # book（テーマ別問数）| serial（日々の演習等・月刊）
themes:               # book のとき
  - { name: 和差算・分配算, total: 10, done: 10 }
  - { name: つるかめ算,     total: 12, done: 6  }
# serial のとき:
# volumes_total: 30
# current: { label: 6月号, unit: 速さと比, total: 32, done: 20 }
# volumes: [100,100,...,62,0,0]   # 各号の達成率
---
```

**テスト結果ノート** `テスト/2026-06-21-復習18-算数.md`
```yaml
---
type: test
kind: 復習テスト        # 復習テスト | 公開テスト
subject: 算数           # 公開テストは「4科総合」
name: 第18回 復習テスト
date: 2026-06-21
range: 速さ・旅人算
score: 82              # 復習テスト=得点
# deviation: 58        # 公開テスト=偏差値
# subject_dev: { 算数: 55, 国語: 60, 理科: 57, 社会: 61 }
reviewed: false        # 見直しずみか
---
```

**間違いノート** `間違い/2026-06-24-速さ-追い越し.md`（本文に問題・解答の写真を貼る）
```yaml
---
subject: 算数
unit: 速さ              # 単元
theme: 追い越し          # 細目（テーマ）
category: テキスト       # テキスト | 復習テスト | 公開テスト
source: 速さテキスト p.32
reason: 考え方のミス      # 計算ミス/問題の読みまちがい/おぼえていない/考え方のミス/時間切れ/うっかりミス
question: 兄が弟を追い越すのにかかる時間
count: 3               # 間違えた回数
date: 2026-06-24
solved: false          # 直しずみか
---
追い越しは『差÷速さの差』。和で割ってしまった。
![[mistake-photo.jpg]]
```

**日々の報告ノート** `報告/2026-06-29.md`（その日 or 翌朝に記入）
```yaml
---
date: 2026-06-29
plan:                  # その日にやること（科目；教材）
  - { subject: 算数, material: プラスワン, done: true }
  - { subject: 理科, material: メモリーチェック, done: false }
study_minutes:         # 科目別の学習時間（学習バランス用）
  算数: 60
  理科: 24
---
今日はプラスワンの速さが難しかった。
```

> **予定（plan）の作り方**: 上のように日次ノートに直接書くか、曜日テンプレート（プロトタイプ `data.js` 末尾のIIFE参照）から自動生成するか、いずれか。学習バランスは `study_minutes` を月内で合算して算出します（問数ベースではなく **時間ベース**）。

## Screens / Views

アプリは左サイドバー＋メイン領域の2カラム。ナビは **ダッシュボード / 間違い直し / テキスト・テスト / 学習カレンダー** の4つ。サイドバー下部に「科目の進捗」（取り組み中の参考書ごとの到達度バー）＋ユーザー情報。

### 1. ダッシュボード（home）
- **目的**: 今日の状況と科目全体の俯瞰。
- **レイアウト**: 上段に「本日の進捗リング（左300px固定）＋ミニ学習カレンダー（右fill）」の2カラム grid。下段に「科目の習得状況」カードグリッド（`repeat(auto-fill, minmax(340px, 1fr))`、gap 16px）。
- **科目カード**: ヘッダー（色ドット＋科目名＋「N 直し／直しなし」バッジ）。本文に **取り組み中の参考書を並列の進捗バーで列挙**（本ごとに独立、直列に繋がない）。各バーに 教材名・本アイコン・今日の進み(+N)・%、下に「N/Nテーマ・N/N問」。下部に **「解き終えた参考書 N冊」** をチップで（デフォルト省略表示）。**ヘッダーに合計%は出さない**（本が増えると下がる違和感を避けるため意図的に排除）。

### 2. 間違い直し（mistakes）
- **目的**: 間違いの記録・見直し。
- **レイアウト**: 科目フィルタ（全科目/算数/国語/理科/社会のピル）＋一覧＋詳細パネル。
- **各行**: 科目色ドット・問題タイトル・出典・回数・日付・「直しずみ／直す」。詳細に 単元／テーマ／カテゴリのバッジ、理由、メモ。
- 追加フォーム（add）: カテゴリ選択（テキスト/復習/公開）→ 科目・単元・テーマ・出典・理由・メモ・写真アップロード2枠。

### 3. テキスト・テスト（materials）
- **目的**: 教材ライブラリ。
- **レイアウト**: カテゴリタブ（テキスト/復習テスト/公開テスト）＋科目ピル。
- **テキスト**: 科目ごとに見出し → 取り組み中の教材カードを `minmax(420px,1fr)` グリッドで。**修了済みの参考書は科目ごとに「解き終えた参考書 N冊」バーで折りたたみ（デフォルト閉、＋ボタンで展開）**。
  - 問題集カード（book）: 全体%・全テーマ/問数・残り問数・今日の進み。テーマ別の進捗バー（科目色で統一）。**完了テーマは「完了したテーマ N件」で折りたたみ（デフォルト閉、＋/−ボタン）**。
  - 日々の演習カード（serial）: 今月号の単元・問数・残り、全30冊の達成棒グラフ、全体%。
- **テスト**: 結果リスト。復習=得点、公開=偏差値（科目ピル選択で科目別偏差値に切替、4科総合は常時表示）。

### 4. 学習カレンダー（calendar）
- **目的**: 「その日に何をやるか」の確認と、日々の報告。学習量の多寡は重視しない。
- **レイアウト**: 上部スリムなタイル3つ（連続学習・今月報告した日・今日の予定件数）→ 日々の報告カード → 月間予定カレンダー → 科目別学習バランス（時間）。
- **日々の報告（DailyReport）**: 「今日／昨日」タブ。その日の予定（科目；教材）をチェックリストで消化、ひとことメモ、「報告する」ボタン。提出後は受領メッセージ＋「修正する」。
- **予定カレンダー（StudyCalendar variant="full"）**: 月グリッド。各日に **科目色ドット＋教材名**（例: 算数＝青の「プラスワン」、社会＝ブロンズの「Z会」）。日曜はお休き(=お休み)、今日は金枠、報告ずみは✓。4科のカラー凡例つき。**学習量ヒートマップ・毎日の学習時間棒グラフは廃止**。
- **学習バランス（SubjectSplit）**: ドーナツ＋横棒。**時間ベース**（「N時間N分」表記、合計時間を中央に）。

## Interactions & Behavior
- サイドバーのナビ／科目進捗バーのクリックで画面遷移（科目クリックは materials をその科目で絞り込み）。
- カテゴリタブ・科目ピルでフィルタ。切替時に lucide アイコンを再生成（本番のアイコンライブラリでは不要）。
- 折りたたみ（完了テーマ／修了済み参考書）はローカル state、デフォルト閉。
- 日々の報告: チェック state＋メモ→送信で受領表示に切替。「今日／昨日」切替で対象日のデータを読み直す。
- トランジション: `--dur-fast`（既定の ease-organic）でホバー背景・スケール。

## State Management
- 現在ページ（`page`）と materials の絞り込み科目（`matSubject`）。
- materials: カテゴリタブ・科目フィルタ・各折りたたみの開閉。
- calendar: 報告対象日（今日/昨日）・チェック配列・メモ・送信済みフラグ。
- データ取得: 本番では `data.json`（Obsidian由来）を読み込み、`subjectStats` 等の集計はクライアント or ビルド時。report送信は Obsidianノートへの書き戻し or 別ストアへ（要件次第。読み取り専用ならエクスポート用の控えとして保存）。

## Design Tokens
配色は `tokens/colors.css`（保管庫ルートの `styles.css` 経由）。主要値:

**科目カラー（重要・この割り当てを踏襲）**
- 算数: `var(--chart-7)`（青系 / slate-blue）
- 国語: `var(--chart-2)` = `#BC6E4F`（テラコッタ）
- 理科: `var(--chart-1)` = `#5E8C88`（verdigris / 緑）
- 社会: `#A9763C`（深めのブロンズ。基調の金と差別化するため固有指定）

**基調・中立色（`tokens/colors.css` 参照）**: `--ink` / `--ink-soft` / `--ink-faint`（テキスト3階調）、`--surface` / `--surface-sunken`（面）、`--line` / `--line-strong`（罫線）、`--gold` / `--gold-deep` / `--gold-veil`（アクセント＝アンティークゴールド）、`--success`（達成・緑）。
- 角丸: `--radius-sm` / `--radius-md` / `--radius-pill`
- タイポ: `--font-display`（見出し）/ `--font-body`（本文）。数値は `.tnum`（等幅数字）クラス。
- スペーシング: カードgap 16〜22px、グリッド `minmax(340〜420px, 1fr)`。

正確な hex・全変数は 上位の `styles.css` / `tokens/` を参照。本番ではコードベース既存のトークン体系にマッピングしてください。

## Assets
- アイコン: lucide（`book` / `calendar-days` / `check-circle-2` / `flame` / `sparkles` / `send` / `plus` / `minus` 等）。本番はコードベースのアイコンライブラリで代替可。
- 間違いノートの問題・解答写真: Obsidian保管庫内の添付画像（`![[...]]`）を参照。
- フォント: `tokens` / `styles.css` の `@font-face` 定義を確認。

## Files
プロトタイプ一式は本READMEの **1つ上の階層 `kurumi-dashboard/` 直下** にあります:
- `../index.html` — エントリ（スクリプト読込順）
- `../data.js` — **モックデータ＋集計ロジック＋カレンダー生成**（データモデルの設計図。最重要）
- `../App.jsx` — シェル（サイドバー・ルーター・科目進捗）
- `../DashboardScreen.jsx` / `../MaterialsScreen.jsx` / `../MistakesScreen.jsx` / `../AddMistakeScreen.jsx` / `../CalendarScreen.jsx` / `../StudyCalendar.jsx`

- デザインシステム本体（`components/`・`tokens/`・`styles.css`）はプロジェクトルート参照。

> 開発者向け: まず `../data.js` を読み、Obsidian frontmatter スキーマ（本README上部）との対応を確認 → パーサで `data.json` を生成 → `MOCK` 参照を差し替え、の順が最短です。
